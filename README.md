## Part 01. 프로그래밍과 친해지자

### 프로그래밍

다른 누군가 미리 설계하지 않은 내 머릿속의 새로운 작업을 기기에게 등록

컴퓨터에 부여하는 명령을 만드는 작업

### 프로그래밍 언어

1. 컴퓨터에게 일을 시키기 위한 사람과 컴퓨터간 소통 수단
2. 컴퓨터 프로그램을 만들때 사용하는 언어
3. 난이도 및 기능성 : 자연어(High Level) << 기계언어(Low Level)
4. Low Level로 갈수록 어려우며 잘 다룰수록 고수

# Low Level Language(피라미드)

## **High Level Language**

- 어셈블리어처럼 기계를 직접적으로 통제할 수 있는 상대적으로 저수준에 가까운 언어
- 하드웨어를 직접적으로 통제하고, 기계의 효율성을 최대화해 성능을 끌어올리기위한 분야에서 많이 사용
- **C++, C#, Java등**  프로그래머가 컴퓨터의 세세하고 복잡한 부분까지 통제하지 않아도 되도록 편의성을 높인 언어
- C++: C언어가 발전된 형태, 하드웨어가 중요하면서도 언어적으로 파워풀한 기능이 필요한 게임,그래픽 등 분야에서 많이 사용
- C# : 마이크로소프트사 개발, 윈도우에서 동작하는 프로그램에 사용
- Java : 전세계에서 압도적으로 많이 사용되는 언어
         모든 기기와 플랫폼에서 동작하도록 만들어진 언어_ 정부제공 사이트 대부분 Java
         언어 완성도 준수, 배우기 어렵지 않고, 다양한 프로그래밍 언어가 공유하는 기본기
         갖추어 Java 알고 있으면 다른 프로그래밍 언어를 배울 때 어려움이 없음
- Python: 자바보다 문법 간결, 초보자 배우기 수월, 진입 장벽 낮음, 
             여러 플랫폼에서 다양한 기능 수행 가능 , 적은 코드로 빠르게 프로그래밍 가능
- Ruby: 웹 제작에서 많이 사용, Python의 라이벌로 인식
- php: 과거부터 웹, 특히 개인 웹사이트에 많이 사용, 게시판 페이지 대부분 php로 작성, 쉽고 간단해 가벼운 사이트를 빠르게 만들 수 있지만 규모가 커지고 보안 문제가 중요해질수록 언어자체의 기능적 한계가 있어 기업레벨 웹사이트에선 잘 사용하지 않음
- Objective-C: 맥, 아이폰, 아이패드 등 IOS 앱을 만들때 사용
- Swift: 애플이 자체적으로 만든 언어, 파이선이나 루비처럼 간결하고 쉬운 언어, 애플 제품내 소프트웨어를 만들때 사용
- Kotlin: 안드로이드용 언어, Swift와 비슷한 문법, 자바보다 간결하고 자바 환경에서도 작동 가능, 구글이 안드로이드 공식 언어로 인증
- JavaScript: 브라우저에서 돌아가도록 개발된 언어(사파리, 크롬, 파이어 폭스 등)
                 서버에서 이뤄지는 작업이 아닌 브라우저상의 팝업창, 애니메이션, 링크 이동 등             
                 제한적인 기능들을 프로그래밍하는데 사용되는 언어로 따라서 과거 프로그래밍   
                 언어로 취급 X / 브라우저에 갇혀있던 스크립트를 컴퓨에서 직접 돌아갈 수 있도
                 록 만든 node.js 등장함과 동시에 JS도 프로그래밍언어로서 입지를 갖게됨

## Assemble Language: (Low Level)

- Machine Language 단점 보완하기 위해 나온 언어
- 기기마다 어셈블러라는 도구를 통해 기계어로 변환 but 장황하고 비 생산적인 언어

## **Machine Language** : (Low Level)

- 컴퓨터 CPU가 그 자체로 알아들을 수 있는 언어 / 0,1로 구성 > 로직 설계 보단 0,1로 변환하고 실수를 확인하는 시간이 더 길음
- 컴퓨터마다 기계어가 달라 장치가 바뀌면 새로운 언어를 배워야했음

## 개발 용어

#### 서버 Server

데이터 제공하는 곳 (Client=데이터 요청) 상대적 개념

1. 제공자 역할을 하는 컴퓨터
2. 컴퓨터를 제공자로 만드는 프로그램

ex, 

1. 서버에 문제 생겼네요 >> 서버컴퓨터의 소프트웨어오류
2. 컴퓨터에 **서버 좀 깔게요** >> 컴퓨터를 서버로 만들어줌, 저장된 정보를 외부로 보내거나 외부에서 들어온 요청에 따라 특정 작업을 통해 보내줄 수 있는 소프트웨어

#### 프론트엔드

눈에 보이는 인터페이스, 디자인 등 시각적 요소 개발,  데이터 받아 스크린에 보여주는 것 / 클라이언트 개발자

HTML,CSS, JavaScript를 이용하여 시각적인 뼈대를 갖추는 퍼블리싱 수행

#### 백엔드

눈에 보이지 않는 뒤에서 처리/ 데이터, 내용상 문제/ 서버에서 처리, 서버 개발자라고도 함 

#### 브라우저

웹 >> html, css ,js로 구성된  파일을 원격으로 가져와 화면에 보여주는 사파리, 크롬, 웨일 같은 프로그램

#### 호환성

브라우저에 따라 기능 작동방식이 달라짐

웹 표준: 다양한 웹 브라우저에서 같은 기능 수행

웹 호환성: 다양한 종류의 브라우저에서 웹이 일관성있게 보이도록 하는 것 

---
#### 네이티브앱

앱의 기능을 각 os마다 따로따로 제작한 제품 

장점: 각 os에 맞게 제작되어 해당 환경에 최적화, 성능 파워풀

단점: 각 os따로 제작해야하기때문에 개발을 두번씩 해야함

       업데이트하려면 새로운 버전을 스토어에     올리는데 검토과정을 거치기때문에 기간이 오래걸림 (검토 및 배포과정을 마친 후 적용되는 시간이 달라 변화를 일괄적용하기 힘듬/ios는 특히 몇일 더 걸림)

#### 웹앱

네이티브앱으로 뼈대만 만들고 내용은 웹사이트로 제작 

특정 사이트를 열 수 있는 모바일 브라우저

장점: 네이티브는 브라우저 역할만 수행하고 실질적 콘텐츠는 웹사이트이기 때문에 서버에 올라와잇는 내용만 변경하면 언제든지 서비스 변경 가능 (개발 운용측면에서 효율적)

단점: 브라우저에서만 도는 웹사이트로 고성능이 필요한 작업에서는 제약
---

#### 적응형

브라우저에서 웹에 접근할때 디바이스를 인식해서 모바일의경우 모바일 주소로 리디렉션

PC와 모바일에서 화면 구조와 컨텐츠가 크기가 많이 바뀌는경우 사용

#### 반응형

하나의 페이지가 스크린의 크기에따라 유동적으로 바뀜

내용이 복잡하지않고 콘텐츠 배치만 달라지는 가벼운 페이지는 반응형으로 제작

### 쿠키

특정 정보가 사용자의 브라우저에 저장

조작되거나, 공개되어도 문제없는 정보

Ex, 마트: 내가 직접 끌고다니는 카트

### 세션

서버에 저장, 사용자 로그인시 사용자 정보를 서버에 저장

사용자가 임의로 조작해서는 안되는 정보, 보안, 결제관련 정보

클라이언트 많아질수록 서버에 부하

Ex, 로그인, 마트: 마트에서 가지고있는 회원정보

시간과 통신데이터를 아끼기 위해 한번 호드한 데이터를 재사용하는 것
